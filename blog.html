Effect of Measurement uints on Html element

The length CSS data type represents a distance value. Lengths can be used in many CSS properties, such as width, height, margin, padding, border-width, font-size and so on. There are mainly two types of Measurement uints. those are relative and absolute length unit.lets see each one by one, before gonna to the effect on responsive ness of the html elemnt on diffrent mediums(devices).
Relative unit Measurement: In this type of units the Measurement represent interms of the other distance i.e Font size of the parent container, length of view port of the actual devices. again, if we want represent a length interms of fontsize of the parent, we can use like em , rem , ex .... and if we want in viewport ,we can use vh , vw , vmin... instead.
absolute unit Measurement : In this type of units the actual Measurement will represents with out considering the actual devices we use. i.e if we represent 10cm or 200px , when we it will represet as we write with out responsiveness.
Now, let's back on the effect of those on responsive. the responsiveness of the html element is on the hand of the two Measurement units. because as i said earlier when we represent the length of the html element in relative unit it will responsive, since it will use the parents properties regaring the viewport of the device. when we use absolute unit we can't get responsive website due to screen size of the devices. In conculsion, the use of relative and absolute Measurement units matter on the responsiveness of the website.
